Service Container13 / 16
Service Container — أداة Dependency Injection
التعريف الرسمي — Laravel Docs

أداة قوية لـ إدارة dependencies الكلاسات وحقنها تلقائياً. بدل ما تكتب new Mailer(new SmtpTransport()) في كل مكان — تكتب اسم الكلاس فقط وLaravel ينشئه لك.

✨ Zero Configuration Resolution

إذا الكلاس ما عنده dependencies معقدة — Laravel يحلّه تلقائياً بدون أي تسجيل. أغلب وقتك ما رح تحتاج تتعامل مع Container مباشرة — هو يشتغل في الخلفية.

متى تتعامل معه يدوياً؟
→ لما تبغى تربط Interface بـ implementation معين
→ لما تبغى خدمة واحدة فقط طول عمر الطلب (singleton)
// Automatic Injection — الأكثر استخداماً

// Laravel يحقن Request تلقائياً
use Illuminate\Http\Request;

Route::get('/search', function (Request $request) {
  // Container حقن Request تلقائياً هنا
  return $request->input('q');
});

// نفس الشيء في Controller
class ProductController {
  public function store(Request $request) {
    // $request جاهز — لم تكتب new Request()
  }
}
// Manual bind — في AppServiceProvider

// ربط Interface بـ Implementation
$this->app->bind(
  PaymentInterface::class,
  StripePayment::class
);

// الآن كل مكان يطلب PaymentInterface
// يحصل تلقائياً على StripePayment
حسب الـ docs الرسمية: "في أغلب الأحيان لن تحتاج للتعامل مع Container مباشرة — هو يعمل في الخلفية تلقائياً." هذا المفهوم سيصبح أوضح أكثر عندما نبدأ Controllers وDependency Injection؛ حالياً يكفي أن تفهم أن Laravel يدير إنشاء الكلاسات تلقائياً.

AppServiceProvider — Laravel 1214 / 16
AppServiceProvider — بوابة تهيئة التطبيق
ما هو؟

يعمل مرة واحدة عند تحميل التطبيق — قبل أي طلب.
هنا تسجّل الخدمات وتضع إعدادات Bootstrap.

register()
سجّل الخدمات في Container
يعمل أول — قبل boot
لا تستدعي خدمات هنا
$this->app->bind(...)
boot()
إعدادات بعد تسجيل كل الخدمات
يعمل بعد register الكل
يمكن استدعاء أي خدمة
Model::preventLazyLoading()
🆕 Laravel 12 — تغيّر مهم
RouteServiceProvider حُذف ← Routes في bootstrap/app.php
AppServiceProvider أصبح المكان الوحيد ✓
app/Providers/AppServiceProvider.php

namespace App\Providers;

use Illuminate\Support\ServiceProvider;
use Illuminate\Database\Eloquent\Model;
use App\Services\PaymentService;
use App\Contracts\PaymentInterface;

class AppServiceProvider extends ServiceProvider
{
  /**
   * 1️⃣ register() — يعمل أولاً
   * سجّل الخدمات في Container هنا
   */
  public function register(): void
  {
    $this->app->bind(
      PaymentInterface::class,
      PaymentService::class
    );
  }

  /**
   * 2️⃣ boot() — يعمل بعد register الكل
   * ضع إعدادات التطبيق هنا
   */
  public function boot(): void
  {
    // منع lazy loading في dev
    Model::preventLazyLoading(
      ! app()->isProduction()
    );
  }
}
