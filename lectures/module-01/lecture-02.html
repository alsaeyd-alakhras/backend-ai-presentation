Constructor & $this
Constructor

دالة خاصة اسمها __construct — تُنفَّذ تلقائياً فور ما تكتب new. استخدامها لضبط القيم الأولية عند إنشاء الـ Object.

$this

مرجع للـ Object الحالي نفسه. بتستخدمه داخل الكلاس للوصول لـ properties ودوال الكلاس الحالي.

$this->name  =  "Ahmad"
Destructor — ذكر سريع

__destruct تُنفَّذ عند تدمير الـ Object. نادر الاستخدام في Laravel — اذكره وكمّل.

PHP

class Order {
  public string $product;
  public int $qty;
  public float $total;

  // يعمل تلقائياً عند new Order()
  public function __construct(
    string $product,
    int $qty,
    float $price
  ) {
    // $this = هذا الـ object
    $this->product = $product;
    $this->qty = $qty;
    $this->total = $qty * $price;
  }
}

// إنشاء Object مع قيم فورية
$o = new Order("Laptop", 2, 999);
// $o->total = 1998
Dependency Injection يعني أن الكلاس يستقبل الخدمة التي يحتاجها من الخارج بدل ما ينشئها بنفسه. مثال سريع: class OrderController { public function __construct(private OrderService $service) {} }

PHP

class Animal {
  public function __construct(
    public string $name
  ) {}

  public function speak(): string {
    return "{$this->name} makes a sound";
  }
}

class Dog extends Animal {
  public function speak(): string {
    // parent:: لاستدعاء الأب
    $base = parent::speak();
    return $base . " — Woof!";
  }
}

$dog = new Dog("Rex");
// "Rex makes a sound — Woof!"

Abstract Classes8 / 15
Abstract Classes
ما هي؟

كلاس لا يمكن إنشاء Object منه مباشرة. يحدد هيكل الكلاسات الأبناء ويجبرها على تطبيق دوال معينة.

متى نستخدمها؟
✓ لما تبغى تحدد عقد مشترك بين عدة كلاسات
✓ لما عندك كود مشترك بين الكلاسات — ضعه في الأب
✓ لما تبغى تجبر الأبناء على تطبيق دالة معينة
PHP

// لا يمكن: new Shape() → خطأ!
abstract class Shape {

  // كود مشترك — يرثه الجميع
  public function describe(): string {
    return "I am a shape";
  }

  // يجب على كل ابن تطبيقه
  abstract public function area(): float;
}

class Circle extends Shape {
  public function __construct(
    private float $r) {}

  // إلزامي التطبيق هنا
  public function area(): float {
    return M_PI * $this->r ** 2;
  }
}
في Laravel مثال: Mailable هي abstract class — كل mail ترث منها وتطبق build().

مجموعة دوال تستطيع مشاركتها بين كلاسات متعددة بدون وراثة. تحل مشكلة عدم إمكانية الوراثة من أكثر من كلاس.

PHP

// trait = كود مشترك قابل لإعادة الاستخدام
trait HasTimestamps {
  public function createdAt(): string {
    return date("Y-m-d H:i:s");
  }
}

trait SoftDeletes {
  public function softDelete(): void {
    $this->deleted_at = now();
  }
}

class Post extends Model {
  // استخدام أكثر من trait معاً!
  use HasTimestamps, SoftDeletes;
}

PHP

class Math {
  // static = لا تحتاج new Math()
  public static function square(
    float $n
  ): float {
    return $n ** 2;
  }
}

// استدعاء مباشر بدون new
Math::square(5); // 25

// مثال أكثر واقعية
class DB {
  private static $instance;

  public static function connect(): static {
    if (!static::$instance) {
      static::$instance = new static();
    }
    return static::$instance;
  }
}
في Eloquent: User::find(1), User::where(...), User::create() — كلها static methods!

Namespaces & PSR-4
لماذا Namespaces؟

تخيل ملفين مختلفين فيهم كلاس اسمه User — الـ Namespace يحدد أي User تقصد. مثل مجلدات الملفات: اسم الملف واحد لكن المسار مختلف.

PSR-4 — القاعدة الرسمية
المجلد: app/Models/User.php
↕
Namespace: App\Models\User
المجلد = Namespace بشكل تلقائي

// ملف: app/Models/User.php

الفكرة

في آخر 5 دقائق نجمع أهم مفاهيم OOP في مثال واحد: سلة مشتريات بسيطة بدون قاعدة بيانات.

ما الذي نستخدمه؟
Constructor لتمرير وسيلة الدفع عند إنشاء السلة
$this لحفظ المنتجات داخل الكلاس الحالي
Interface لتبديل طريقة الدفع بدون تغيير السلة
Trait لإضافة Logger بسيط يعرض خطوات التنفيذ
PHP

interface PaymentGateway {
  public function charge(float $amount): bool;
}

trait LogsActivity {
  public function log(string $message): void {
    // echo $message;
  }
}

class ShoppingCart {
  use LogsActivity;
  private array $items = [];

  public function __construct(
    private PaymentGateway $gateway
  ) {}

  public function add(string $name, float $price): void {
    $this->items[] = ['name' => $name, 'price' => $price];
    $this->log('Item added');
    $this->gateway->charge($price);
  }
}
هذا المثال ليس مشروعاً كاملاً — فقط تجميع سريع للمفاهيم قبل أن نرى نفس الأفكار داخل Laravel.
